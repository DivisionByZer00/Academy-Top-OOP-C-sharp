## Homework 01

### Тема: "От классов к архитектуре. Мышление объектами"

**Задание:** Вам необходимо создать простую систему для учета товаров на небольшом складе (инвентарь). Программа не будет иметь интерактивного меню, ее задача — продемонстрировать правильное проектирование моделей данных и класса, который ими управляет.    
**Логика работы программы**
- **Модель Product:** Вы должны спроектировать и реализовать record для представления товара. Он должен хранить уникальный артикул (ID), название и цену.
- **Класс Inventory:** Вам нужно создать класс, который будет управлять списком товаров. Этот класс должен инкапсулировать (скрывать) в себе коллекцию List<Product> и предоставлять публичные методы для работы с ней.
- **Демонстрация в Main:** В основном методе программы вы должны продемонстрировать работу вашей системы: создать инвентарь, добавить в него несколько разных товаров, а затем найти один из товаров по его артикулу и вывести информацию о нем.

## Homework 02

### Тема: "Наследование. Построение иерархий и переиспользование кода"

**Задание:** Спроектировать и реализовать простую модель для расчета заработной платы разных типов сотрудников в компании. Необходимо построить иерархию классов, где Manager и Developer являются наследниками общего класса Employee, и продемонстрировать полиморфную работу с
ними для расчета и вывода зарплаты каждого сотрудника..    
**Логика работы программы**
- **Базовый класс Employee:** необходимо реализовать базовый класс для всех сотрудников. Он должен содержать общие свойства Name и BaseSalary, а также виртуальный метод CalculateMonthlySalary(), который по умолчанию возвращает базовую зарплату.
- **Производный класс Manager:** необходимо реализовать класс, наследуемый от Employee. Он должен иметь дополнительное свойство Bonus и переопределять метод CalculateMonthlySalary() для учета этого бонуса.
- **Производный класс Developer:** необходимо реализовать класс, наследуемый от Employee. Он должен иметь дополнительное свойство LinesOfCodeWritten и переопределять метод CalculateMonthlySalary() для расчета премии на основе написанного кода.
- **Демонстрация в Main:** в основном методе программы вы должны продемонстрировать полиморфную работу системы: создать список List<Employee>, добавить в него объекты Manager и Developer, а затем в цикле вывести зарплату для каждого, вызывая их индивидуальные версии метода CalculateMonthlySalary().
