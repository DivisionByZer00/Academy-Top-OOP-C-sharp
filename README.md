Когда новый участник делает ставку на лот, AuctionLot должен уведомить всех остальных участников (кроме того, кто сделал ставку) о том, что цена изменилась. Это классический пример паттерна "Наблюдатель", который вы должны реализовать с помощью механизма событий C#.

3. **Демонстрация в Main:** В основном методе программы вы должны:
- Создать один экземпляр AuctionLot.
- Создать несколько экземпляров Bidder.
- Подписать всех участников на событие лота.
- Сымитировать торги, вызывая метод PlaceBid от имени разных участников и наблюдая, как остальные получают уведомления.

### Тема: "Сериализация объектов и сохранение состояния"

***Задание:*** Вам предстоит создать небольшую консольную программу, которая имитирует сохранение и загрузку настроек приложения. Вы создадите класс для хранения настроек и класс-менеджер, который будет отвечать за их сериализацию в JSON-файл и десериализацию обратно в объект.    
Ваша задача — продемонстрировать, что состояние объекта Settings может быть успешно сохранено на диск и восстановлено при следующем "запуске" программы.

***Логика работы программы***
- **Класс Settings (Продукт):** Вам нужно создать простой класс, который будет хранить различные настройки. У него должны быть свойства, например: VolumeLevel (int), Language (string), IsDarkMode (bool).
- **Класс SettingsManager:** Вам нужно создать класс, который будет отвечать за логику сохранения и загрузки. Он должен иметь два метода:
  - void Save(Settings settings, string filePath): Сериализует объект Settings в JSON и сохраняет его в файл по указанному пути.
  - Settings Load(string filePath): Читает JSON из файла, десериализует его в объект Settings и возвращает его. Если файл не существует, метод должен возвращать новый объект Settings со значениями по умолчанию.
- **Демонстрация в Main:** В основном методе программы вы должны продемонстрировать полный цикл работы:
  - Создать объект Settings со значениями по умолчанию.
  - Сохранить его в файл с помощью SettingsManager.
  - Загрузить настройки из этого файла в новый объект.
  - Изменить какое-либо свойство у загруженного объекта.
  - Сохранить измененные настройки обратно в тот же файл.
  - Загрузить настройки еще раз в третий объект, чтобы убедиться, что изменения были успешно сохранены.

## Homework 23

### Тема: "Подготовка к защите финального проекта (и к мировой славе)"

***Задание:*** Вы только что завершили свой проект "Мини-CRM", и он настолько хорош, что вас номинировали на Нобелевскую премию в области объектно-ориентированного проектирования. Ваша задача — написать краткую, но пафосную благодарственную речь для церемонии вручения.    
Это творческое задание, которое поможет вам в свободной форме вспомнить и с юмором отрефлексировать все, что вы изучили.

***Логика работы (написания речи)***
- Вам нужно создать простой текстовый файл и написать речь (5-10 предложений), которая в шутливой форме упоминает несколько концепций из нашего курса.
- О чем можно упомянуть в речи (на выбор):
- SOLID: Поблагодарите Барбару Лисков за то, что ее принцип не дал вашим квадратам сломать прямоугольники. Или упомяните, что ваш код настолько SOLID, что им можно колоть орехи.
- Паттерны: Расскажите, как вы построили "Мост" к успеху, использовали "Фасад", чтобы скрыть гениальную сложность, или как "Одиночка" помог вам не чувствовать себя одиноко в процессе разработки.
- Рефакторинг: Упомяните, что вы оставили код "чище, чем он был до вас", следуя правилу бойскаута.
- Технический долг: Заявите, что в вашем проекте нет технического долга, а есть только "архитектурные инвестиции в будущее".
- Тесты: Похвастайтесь, что ваш Test Explorer настолько "зеленый", что его можно использовать для фотосинтеза.
